이 저널은 GitHub Pages에 잘 맞는 파일 기반 구조로 동작합니다. 게시글 하나는 메타데이터 1개와 본문 파일 1개로 구성됩니다.

무엇이 바뀌었나

  • tab_4.html이 게시글 목록을 자동으로 렌더링합니다.
  • blog-post.html이 선택한 slug 기준으로 본문 전체를 보여줍니다.
  • assets/data/blog-posts.json이 제목, 날짜, 요약, 이미지, 파일 경로를 관리합니다.

새 글 추가 방법

  1. 커버 이미지를 images/blog 또는 원하는 이미지 폴더에 넣습니다.
  2. content/blog/_template.html을 복사해서 새 글 파일 이름으로 바꿉니다.
  3. assets/data/blog-posts.json에 새 항목을 추가합니다.
  4. 변경사항을 GitHub Pages 저장소에 푸시합니다.
이 구조는 정적 GitHub Pages 사이트이므로, 글 작성과 삭제는 저장소 파일을 수정하는 방식으로 이루어집니다. 대신 페이지는 가볍고 유지보수는 단순해집니다.
