This journal uses a simple file-based workflow that fits GitHub Pages well. Each post is made from one metadata entry and one content file.

What changed

  • tab_4.html now renders a post list automatically.
  • blog-post.html displays the full article for a selected slug.
  • assets/data/blog-posts.json controls titles, dates, excerpts, images, and file paths.

How to add a new post

  1. Put a cover image inside images/blog or any image folder you prefer.
  2. Copy the template file in content/blog/_template.html and rename it.
  3. Add a new object to assets/data/blog-posts.json.
  4. Push the changes to GitHub Pages.
Because this is a static GitHub Pages site, posting and deleting still happens by editing files in the repository. The public page stays lightweight, and maintenance stays simple.
